# pgrep L5.9 — UX punch list (ship polish)

A ship-oriented sweep of the six desktop surfaces. Each item names the file, the
owner lane, and the fix. The content and AI lane items are handled on their own
branches; the rest are UI-agent (single-writer) work that Frank routes to the P2
UI branch. Nothing here is a blocker; the app runs and scores honestly AI-off
today. These are the polish gaps between "works" and "ships".

## Done (content and AI lane)

- **Library math rendering.** Sibling and saved card fronts now typeset delimited
  LaTeX through the shared `renderMath` (adopted, not forked), matching CardFace.
  `ts/routes/pgrep/library/+page.svelte`. Landed on `l5.9-library-math`.

## UI-agent quick wins (small, high visibility)

1. **Progress "Scores" tab omits the Memory card.** Home shows all three score
   cards; the Progress Scores tab shows only Performance and Readiness (Memory is
   just an inline % in the coverage list). Add the Memory `ScoreCard` for parity.
   `pgrepMemoryScore` already exists and is wired on Home.
   File: `ts/routes/pgrep/progress/+page.svelte`.

2. **Settings sync URL is not persisted** and its default can mismatch the sync
   server. Persist the field and align the default to the port `just sync-server`
   listens on (8080, per `l5.9-decisions.md` E1).
   File: `ts/routes/pgrep/settings/+page.svelte`.

3. **Theme-token leak.** The armed reset button uses a raw `color: #fff` instead
   of a token. Swap for the on-action token.
   File: `ts/routes/pgrep/settings/+page.svelte` (armed-reset style, ~line 606).

4. **Study topic chip shows a raw slug** (`mechanics::dynamics_energy`) where Exam
   and Diagnostic already show a human label. Reuse the same slug-to-label
   formatter.
   Files: `ts/routes/pgrep/study/+page.svelte`, `ts/lib/components/StudyFrame.svelte`.

5. **Exam answer failures are silent.** A dropped `pgrepExamAnswer` fails without
   a toast; the run recovers but the user gets no signal. Add a quiet retry or a
   non-blocking notice.
   File: `ts/routes/pgrep/study/exam/+page.svelte`.

## Bigger UX (needs a backend read-model, so coordinate)

6. **Home manifold hero is decorative, not live.** The knowledge-map uses a static
   `FULL_SURFACE` constant; nothing feeds it from the learner's real coverage or
   scores, which undercuts the "shown honestly" headline. Wire it to a small
   backend read-model over the existing scores/coverage.
   Files: `ts/routes/pgrep/+page.svelte`, `ts/lib/components/Manifold*.svelte`,
   `ts/lib/pgrep/manifold*.ts`, plus a new `pgrepManifold*` bridge handler.

7. **Exam figures not served.** The exam UI has a figure slot, but the exam read
   model does not carry a diagram. The 38 bundled figures already live on the
   problem stems, so this is wiring the exam read model to pass the stem figure
   through. Split: content/AI lane owns the figures and the bundle; UI agent owns
   the exam surface; `pylib/anki/pgrep/exam.py` is the seam (route through Frank).

8. **Diagnostic quick-checks are hardcoded in the frontend** (`QUICK_CHECKS`) and
   plain text (no LaTeX). Move the content to a backend endpoint and render it
   through `renderMath` so it can change without a frontend deploy.
   File: `ts/routes/pgrep/diagnostic/+page.svelte` plus a bridge handler.
