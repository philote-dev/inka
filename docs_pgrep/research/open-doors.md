# Open Doors — future feature ideas

A standalone scratchpad for feature ideas Frank wants to remember. Intentionally
not wired into the doc map and not cross-linked. This is NOT a backlog of
build-plan work or project requirements. It holds only genuine "someday, maybe"
feature notes that are not otherwise scheduled. Jot freely. Promote an item into
a real design doc only when we decide to build it.

## Features

- User-provided source upload (bring your own textbook or notes) as extra RAG grounding for generation.
- Learn / remediation surface: full teaching for genuinely cold topics (teaching fallback).

## Models (only once real users / more data exist)

- Knowledge tracing (DKT / AKT).
- IRT / Elo item calibration (needs many users per item).
- Longitudinal validation of the score models with real cohort data over time.
- Confidence-capture calibration extras: only relevant if we ever start asking the user to predict their own correctness. Parked because we deliberately do NOT capture user confidence today. (Examples: targeting high-confidence errors, using response time to flag low-effort answers, timed-vs-untimed review comparison, a no-penalty guess-strategy trainer.)
